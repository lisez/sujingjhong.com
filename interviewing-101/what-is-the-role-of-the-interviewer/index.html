面試官新手指南：面試官是什麼樣的角色

  • 📝 發布日期 2024-08-02
  • 📅 更新日期 2024-08-02
  • 🔤 字數 3323

現在我該做甚麼？ #

角色，依教育部辭典有兩種定義：

  1. 戲劇中演員所扮演的劇中人物。有主角、配角之分。
  2. 一個人在團體中，依其地位所擔負的責任或所表現的行為。

一開始擔任面試官，我第一個反應是：不知所措。鎮定地參與面試流程，專心地聽完面試者問答，盡我所能擠出問題。其他時間多在觀察同組面試官反應。因為我還不知道面試官究竟要幹嘛。

友人也有分享面試故事。她抵達公司時，一位年齡相仿職員熱情招呼，請她填寫個人資料表，接著簡單提問。不久後，用人主管到來，就切入正題詢問專業智識、文化適性、職涯發展等等問題，兩位氣場迥隔霄壤，逐漸招架不住。面試完後認為應該沒機會了。幾天後意外收到錄取通知。用人主管表示其中一個原因是那位同齡職員很喜歡她，這才意識到原來她也是面試官。兩位氣勢天差地遠。

聽到這則故事，我完全可以理解。那時我就是位稚嫩面試官。完全不懂面試場域中面試官職責為何，只能靜觀其變，看看其他面試官表現、收集面試者心得側寫，才能慢慢地、片段地了解面試官這個身分。

言歸正傳，出任面試官首要第一步，就是要了解它在面試場域中的職責與舉措。

面試官與非面試官間角色切換 #

首先要確定是，面試官只是一種暫時性職務、一種任務編組或者是臨時性需要職務。面試官在沒有面試時，就只是一般公司職員，需要進行日常工作。面試官是因應公司用人需要，依管理階層需求編組，於任務結束或一定時間後解除。所以這裡將面試官與一般職員身分做比較，從差異去了解角色。

從對事到對人 #

除非你是老闆或股東，不然大多數時間受僱者多半是執行者，可能是來自客戶需求、跨部協作或者是主管交辦事項，執行事項通常是事物性任務，特徵上有著一定目標性，例如完成一個登入系統：或者是時效性，例如三天內修復客戶回報的障礙缺陷；以及會交辦到身上的，通常具有可執行性，其他人認為是你在能力範圍內足以勝任完成。

即便是主管職務有部分時間在處理人事性任務，但多半也是圍繞在：某人執行某任務時所發生的問題。盡量避免對個人本身進行檢討，盡量抽離個人情緒、專注在事物。萬不得已，開始檢討工作態度或者個人反應時，離職、資遣已不遠矣。

而面試官工作執行內容就是評斷面試者。

身分轉變到面試官時，甄選人才有著強烈對人性，和日常工作性質大相徑庭。雖然仍具有目標性：找到人；有時間性：盡快；也有可執行性：要錄取還是不要錄取。但你就是要針對此人在面試中所為表現作出評價。即使有著作品集或者履歷審查，依舊是避不開針對個人作出評價。

無論面試有幾面，一面、二面、三面甚至看過有公司有九輪面試，一場面試大概 2 小時，用九輪計算也才 18 小時，你就是要在這 18 小時中，針對此人過往 N 年學經歷作出評價。

從解答到提問 #

接著在日常工作中，我們多半在解答問題。現代公司組織結構朝向專業分工細化，職位是具有責任領域。傳統上可能依據產銷人發財架構組織，例如人資部、財務部等。單一部門內會以專業中子領域區分，例如以軟體工程師來說，會依據前端、後端、維運、品管等領域區分責任範圍。

也因為專業分工，跨部門協作時容易出現大哉問：「壞掉了，怎麼辦？」「怎麼報帳？」「這工具要怎麼用？」。在自身職務領域，大多數時間就是在解答或處理這類問題。而即使詢問其他部門時，通常也是大哉問亂鬥。

面試官則是需要設計問題。角色轉換到面試官時，除了預留時間讓面試者提問外，大部分都是面試官提問。若想要在時限內找出適當的面試者，好問題就很重要。好的回答，來自好的提問。面試官必須設計問題、了解問題，也才能對回答作出評價。

評價與決定 #

再來面試官最重要的事情，就是作出評價與決定。面試官須能僅依賴面試者履歷、作品集、問答以及舉止，這些不充足資訊決定：是否錄用面試者。

這決定最弔詭是，你一定必須錄取面試者，面試者必定要經過一段時間後，面試官才有辦法得知錄用決定正確與否；即使知道後，仍不確定每位候選人表現變數為何，不確定到底是人的問題、組織的問題、導師的問題、大環境問題等。此外錄用決定，影響公司與面試者，你可能會改變他人人生軌跡甚至是公司發展方向，這種伯樂與千里馬故事中外可循，就不再贅述。

總結來說，我認為面試官角色就是：對人提問作出評價再加以論斷。對於新手面試官，要怎麼快速進入這角色？我建議就是：確立人設，以及劃定提問範圍。

確立人設與提問範圍 #

定義你的面試形象 #

第一步就是確立人設，定義面試形象，思考面試者對你第一印象為何。

人設是人物設定簡稱。在文學創作中，作者要先決定角色背景、基本資料、優勢缺陷等，再將人物置入場景，讓他舉止有理，讀者才不會出戲。比方說，如果故事中有位貴族世家少爺在衝突中暴粗口，讀者可能會莫名其妙是在哪學到的，這時候就要適當揭露背景，例如長輩說溜嘴、偷聽傭人對話等等，去合理解釋怎麼在日常生活中接觸到粗俗用詞。

在擔任面試官時也是。可以想想：

  • 你希望面試者對你第一印象是什麼？
  • 你想營造出什麼氣場？
  • 你想讓面試進行氣氛如何？

你可以寫下來，並在腦內模擬：當你是面試者時，「面試官」如何和你互動。這就會影響到你在面試場域中穿著打扮、行為舉止、口條措辭等，因為諸如此類外顯行為，來自內在形象投射。

舉例來說，如果人設是一位難以相處的面試官，就會開始打斷對話、咄咄逼人、用詞尖銳；或者想要營造高強度、高壓面試官，在面試氣氛上就會偏向冷洌壓抑、問題連珠炮響、挑眉白眼等等；你也可以做一位溫暖面試官，從噓寒問暖到包容開放。以上都可能會是面試官人設。

你可以有不只一種角色。可以先設定一個角色形象，之後再設定其他形象，協助你應對不同面試者或者職務人選。就像人在與不同生活圈團體互動時，表現多少略有不同。你可以在朋友面前大聲吆喝，但可能回到家是輕聲細語。面試官也是，你可以隨著現場、人選或其他面試官組合不同，派上不同角色。

另外要注意刻板印象與象徵符號。刻板印象是快速構建角色形象工具之一，但同時也是雙面刃。你可能會被指責助長刻板印象，或者附合刻板元素造成形象衝突。在選用各種模板元素中要有意識地注意到。另一個容易衝突則是象徵符號，常見例如十字架、新月在現今已有宗教隱喻，與其他元素搭配時須多加注意；另一種常被疏漏的就是言談中的代稱，有些代稱已經被污名化、標籤化，在面談中要小心使用。

劃定面試提問範圍 #

確定好人設後，第二步就是劃定提問範圍。

思考你在面試場域中的角色，接著問題層面與領域。面試時間有限，一次約莫 1 到 2 小時，如何在時限內用最少問題獲得最多資訊是重要課題。方法上可以粗略地分類成兩種策略：問卷分析和焦點訪談。

問卷分析你應該耳熟能詳，通常就是一坐下或者事先給予面試者一系列問卷填寫，例如個人資料表、適性測驗、智力測驗等等。焦點訪談，就是對特定團體制定一系列問題徵詢對象對主題意見、態度、觀感等等。

無論採取何種方式，都需要事先界定提問主題，才有辦法在時限內有效問答。舉例來說：

  • 想要了解專業知能，那就需要安排測驗或者特定職能領域問題，例如 JavaScript 中 this 和 globalThis 的差異是什麼？
  • 想要了解專業倫理，可能就會出現情境題，例如選用開放原始碼 GPL 授權套件，但會汙染公司商業軟體原始碼，這時候你會怎麼作？
  • 想要了解人格特質，常見是 DISC 或者是 MBTI，去判斷面試者是什麼動物或是 X 人。

一開始如果不確定要怎麼界定範圍時，這裡有兩個建議：

  • 先將所有想到的問題列出，再進行歸類。例如先想辦法擠出 20 個問題，接著為每個問題貼上標籤，開始去聯想每個標籤是否有共同概念。反覆進行幾輪，就可以收斂出你有興致的提問範圍。
  • 收集網路或書籍上面試經驗文或是問題清單成為初始題庫，在從中去蕪存菁，留下關鍵問題。

不再慌張 #

最後記得多和招募小組成員分享意見。他山之石可以攻錯，適時尋求他人建議是必要路徑。記得面試官也代表公司形象，稍不注意就會構成公關災難，觸發炎上事件。所以多和你的面試小組、主管或是上級聊聊，了解他們對於面試流程的看法、了解是否符合公司規章與貼近公司文化，形成共識甚至是角色分工。

希望這篇文章可以讓你瞭解到面試官職務和原職務身分不同處，以及可以參考他人建議去設定自我形象，並確立自己對面試者提問範圍到哪。注意注意面試官身分舉止至關重要，是面試者形成公司印象第一線，稍不注意星星之火足以燎原，多和召募小組成員聊聊形成共識。

我希望按照上面方法，可以讓你在初任面試官時，不再慌張、有所進退。
